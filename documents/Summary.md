# Отчётные документы по итогам автоматизации

## Запланировано
1. Автоматизация тестирования веб-сервиса покупки тура:

  * покупка тура по карте;
  * покупка тура в кредит.
2. Проверка позитивных и негативных сценариев:
  * успешная покупка;
  * покупка по отклонённой карте;
  * валидация полей формы (номер карты, месяц, год, владелец, CVC/CVV).
3. Проверка корректности записи данных в базы данных:
  * MySQL;
  * PostgreSQL.
4. Формирование отчётов о результатах тестирования с помощью Allure,Gradle

## Реализовано
1. Реализованы автотесты для сценариев:
  * покупки тура по карте;
  * покупки тура в кредит.
2. Покрыты позитивные и негативные сценарии:
  * успешные операции;
  * операции по отклонённой карте;
  * некорректные данные в полях формы.
3. Реализована проверка записей в базах данных MySQL и PostgreSQL.
4. Настроена генерация отчётов Allure,Gradle
5. Подготовлены баг-репорты по найденным дефектам.

##  Сработавшие риски

1. Сложности с настройкой и запуском приложения
2. Проблемы с подключением к разным СУБД
3. Недостаток опыта в автоматизации.

## Общий итог по времени

1. Подготовка тестового окружения - планировалось 14 часов, фактически 25 часов. возникли сложности с настройкой двух СУБД.
2. Написание тестовых сценариев - планировалось 10 часов, фактически 6 часов. Часть сценариев была типовой и не потребовала детальной проработки.
3. Автоматизация - планировалось 50 - 60 часов, фактически 70 часов. Недостаток опыта.
4. Отчётные документы по итогам тестирования - планировалось 8 часов, фактически 8 часов. 
5. Отчётные документы по итогам автоматизации - планировалось 8 часов, фактически 3 часа. 
6. Исправление замечаний преподавателя - 12 часов
