# Процедура запуска автотестов

## Бизнес-часть
Приложение — это веб-сервис, который предлагает купить тур по определённой цене двумя способами:

Обычная оплата по дебетовой карте.
Уникальная технология: выдача кредита по данным банковской карты.
<img width="705" height="428" alt="service" src="https://github.com/user-attachments/assets/f51b3205-56df-4f2e-98a1-e311f1c429ca" />

Само приложение не обрабатывает данные по картам, а пересылает их банковским сервисам:

* сервису платежей, далее Payment Gate;
* кредитному сервису, далее Credit Gate.
Приложение в собственной СУБД должно сохранять информацию о том, успешно ли был совершён платёж и каким способом. Данные карт при этом сохранять не допускается.

## Предварительные требования 
Перед запуском автотестов на компьютере должно быть установлено и настроено следующее программное обеспечение:
1. Intellij IDEA
2. Google Chrome
3. Docker
4. DBeaver

## Запуск тестируемого приложения и БД
1. Склонировать репозиторий проекта с помощью команды `git clone <url_репозитория>`
2. Запустить `Docker`
3. В корне проекта выполнить команду для остановки и удаления ранее запущенных контейнеров `docker-compose down`
4. После удаления запускаем контейнеры `docker-compose up`
5. В новом окне терминала запустить тестируемое приложение `java -jar ./artifacts/aqa-shop.jar`
6. Проверить готовность программы к работе по адресу `http://localhost:8080`
7. В третьей вкладке терминала запускаем автотесты для БД MYSQL `./gradlew clean test -Ddb=mysql`, для POSTGRESQL `./gradlew clean test -Ddb=postgres`
8. Для остановки приложения, контейнеров и автотестов в каждом окне терминала нажать комбинацию клавиш `Ctrl+C`
